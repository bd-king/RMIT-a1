My Project

Overview

A business practice that I have experienced as a tradesperson, is time tracking for individual tasks. This is often done by having employees record their hours on each task with some form of badly implemented IT system.

Trades tasks have long been an enigma to those who try to manage them from an administration point of view. The lack of understanding of those who are attempting to gather data for use by management often don't realise the disruptions their policy and hardware implementation creates for the workforce. I would like to put forward a system that satisfies the data gathering requirement with minimal disruption to the tradesperson.

I propose this in the form of a mobile application with a connection to a wirless network, installed in the workshop and a centralised server to collect the data. The server will host a website to both store and manage tasking associated with the mobile application.

Motivation

Many trades multi task througout the day to be efficient across many jobs in a given time frame. An example of this is the Automotive Refinishing industry where paint cure times are a requirement of the product being used. Using a 'clock on job' and 'clock off job' policy that literally forces a paint tradesperson to watch paint dry, is highly inefficient and does not reflect current trade practices. Furthermore, many of the IT systems implemented rely on desktop systems to front the software, causing bottlenecks of tradesmen updating their information.

In my own observation of these job tracking policies I've experienced frustration from myself, from managers of shop floors and from those who have created these systems as they try to understand what we need.

Description

I propose to develop a customised mobile application for tradesmen to use that is primarily tile/icon based to capture data required by management.

Using a mobile based application, tasked employees can quickly update their status with individual jobs using a device at hand, rather than using a desktop based system that may be located some distance from the working location. Workshop level managers will be able to access this collected data to manage the overall picture of workshop requirements. Higher levels of management will be able to access correct data due to the better efficiency. Management software is hosted on a web server and is accessed via a modern browser. This will allow for either a client purchased server to be used or to utilise a cloud based system.

The overall design of the icon based software is as follows:

  • Initial login screen. The use of this screen would depend on the deployment of devices to an employee and the security level requirements of management. A sliding scale of security levels would be implemented for use by management to meet these requirements.
  • Once logged in by the employee, all jobs assigned to that person will show on tiles/icons with an informative number assigned to each tile. For example, an automotive painter could be assigned a cars registration number as a way for management to track jobs.
  • If one job has multiple tasks under it, another screen of icons will display after the initial job tile has been pressed, with all the individual tasks displayed on tiles. Extra levels of tasks could be added creating a tasking tree. Management will create the tasking tree using the website software.
  • Once navigated to the end of the task tree, a screen with a ‘start task’ icon and timer will appear allowing the employee to start work with time tracking enabled. If the employee wishes to multi task and perform multiple jobs at once, they can choose to go back and select more tasks. The final time recorded will then be divided across the amount of tasks currently being tracked.
  • Other tiles could also be implemented for breaks or for specific tasks not related to the tracked job but are still required by the client to be performed by the employee.

The use of an tile/icon based system greatly reduces the amount of time an employee needs to interact with a device. Signing on to a task/s should only take seconds, provided the device required to be used is close at hand.

The management software is website based for managers to create, view and push task data to the appropriate employee, it will utilise a database and server side scripts to achieve this. Reports can be generated and displayed with built in utilities capable of displaying the data as raw reports or with graphs.

Tools and technologies

To achieve this proposal, a number of technologies will be required.

  1. The Mobile application will be coded using Java. This will allow for the best compatibility across many devices.
  2. A wireless network will need to be installed at the intended workshop with as many access points and routing as is required for continuous connectivity. This will then need to be connected the network where the client hosts the server.
  3. A web server will be required to send, receive and store data and to host the website. It will be a Linux based system with Apache, mySQL & PHP installed to cover website requirements. This can either be local or cloud server.
  4. The website will be coded with standard languages such as HTML, JavaScript & PHP.
  5. Hardware such as tablets, smartphones and desktop computers will be used to interact with the software.

A range of skills is needed to create this system. Some of these skills can be outsourced such as the installation of workshop hardware and the creation of the network with wireless access points. There are many companies available to do this.

The critical skills needed to create the proposed software are:

  • A good aptitude with Java and its associated development environments.
  • Website development with a good aptitude in HTML, JavaScript, mySQL and PHP.
  • Skill with deploying a LAMP (Linux, Apache, mySQL and PHP) server.

These technologies have been chosen as they are the standard tools of choice for both web development and mobile application development. They are easily sourced via the internet and there is a wealth of free information online on their use.

Outcome

The initial outcome of this system is an increase in efficient data gathering at workplaces with minimal disruption to workers. Accuracy of data will increase over time creating better management and planning policies for workshop tasking at the tradespersons level.

Proven success of this software would allow it to be further developed into a broader system, potentially it could be used in any workplace that requires time tracking of staff. It could also be developed to connect to existing tracking systems.

There are some human factors that may mitigate the effectiveness of this system.

  • Unwillingness or skill level of tradspersons to use the mobile devices.
  • The system is dependant on managers to create effective tasking.
